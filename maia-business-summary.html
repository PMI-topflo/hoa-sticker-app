01Platform Overview

MAIA (Management & Automated Interaction Assistant) is an enterprise-grade SaaS platform built by PMI Top Florida Properties to digitize and automate HOA homeowner services across 25 associations in South Florida. It provides residents, board members, real estate agents, and vendors with a single omnichannel touchpoint for all property management needs — from paying dues and submitting applications to requesting maintenance and accessing documents.

The platform is built on Next.js 16 (App Router) with Supabase as the backend, deployed on Vercel, and integrates Stripe for payments, Twilio for multi-channel communications, Claude AI for intelligent responses, and ApplyCheck for applicant screening.

03Core Features & Capabilities

🏠 Association Resident Portal

Each of the 25 associations has a dedicated page accessible by homeowners. Pages include links to pay HOA fees, download the PMI Mobile App, submit estoppel requests, and access a Google Drive document library organized into 12 categories (rules, financials, maintenance, board minutes, leases, condo docs, insurance, ACH forms, welcome letters, budget, FAQ, violations).

📋 Resident Application System

A multi-step, multilingual application wizard (EN/ES/PT/FR) handles Individual ($100), Couple ($150–$200), Additional Resident ($100), and Commercial ($100/principal) applications. Includes document upload (government ID, proof of income, marriage certificate), Stripe payment, and ApplyCheck background screening.

💳 Stripe Payment Processing

Checkout sessions created server-side with dynamic line items based on application type. Webhook listener marks applications as paid, triggers confirmation emails to applicants and staff, and fires the ApplyCheck screening flow. All price IDs are environment-configured.

🔍 Applicant Screening

ApplyCheck integration sends screening invitations post-payment. Supports SmartMove (SSN-based) for domestic applicants and an International package. Multi-principal support for commercial entities. Status tracked in the applications table (invited / partial / error).

🤖 AI Omnichannel Communications

A unified webhook handles inbound SMS, WhatsApp, and voice IVR (Twilio). Maia — the AI assistant — identifies callers by phone number across 5 contact tables, maintains multi-turn conversation state in Supabase, and responds in 6 languages (EN, ES, PT, FR, HE, RU). Powered by Claude AI for natural language understanding.

🛠️ Admin Dashboard

Staff-facing dashboard at /admin provides a searchable, filterable, paginated view of 800+ homeowners across all associations. Supports full CRUD (add, edit, delete) via server actions. Includes a New Unit Buyer form with file uploads, document archiving to Supabase Storage, and automatic HTML email notification to the team.

📧 Email System

Centralized email gateway at /api/send-email using Gmail API (OAuth2) via the maia@pmitop.com Google Workspace alias. Includes RFC 2822 MIME construction, module-level token caching, and graceful fallback to Resend. Emails sent for: application confirmations, payment receipts, buyer notifications, escalations.

📁 Document & File Storage

Supabase Storage manages two buckets: application-docs (gov ID, income proof, marriage certificates) and buyer-docs (purchase contracts, ID copies, proof of funds). Files are accessible via signed URLs valid for 30 days. Max file size 10 MB; formats: PDF, JPG, PNG, DOCX.

05Key Business Workflows

New Resident Application

1
Select type — Individual, Couple, Additional Resident, or Commercial Entity
2
Enter applicant details — name, email, DOB, phone; co-applicants as needed
3
Upload documents — government ID, proof of income, marriage certificate (optional)
4
Stripe payment — $100–$200 depending on type; hosted checkout
5
Webhook fires — application marked paid, confirmation email sent, ApplyCheck screening triggered
6
Board review — 7–10 business days; decision sent via email with reference number

New Unit Buyer Notification

1
Staff submits form at /admin/new-buyer with buyer, seller, and agent details
2
Documents uploaded — purchase contract, ID copies, proof of funds to Supabase Storage
3
Email dispatched — HTML notification with signed file links (30-day access) sent to relevant departments
4
Record created — buyer_notification entry stored in applications table for audit trail

10Security & Compliance

🔒Data Security

  • Supabase RLS (Row Level Security) enforced at DB layer
  • Service-role key restricted to server-side only
  • Stripe webhook signature verification (HMAC)
  • Internal API endpoints protected by x-internal-secret header
  • Signed URLs for file access (30-day TTL)
  • SSN / financial data handled exclusively by ApplyCheck (never stored locally)

📜Compliance Features

  • Florida Statute reference field per association
  • Board approval workflow for all resident applications
  • Invoice approval system via CincWebAxis portal
  • Applicant consent checkbox required before payment
  • Full conversation audit trail in Supabase
  • Estoppel certificate workflow (Condocerts) for property transactions
  • ACH authorization forms per association
